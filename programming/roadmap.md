# Programming roadmap (6 LAYERS)

## 1. HỆ THỐNG 6 TẦNG KIẾN THỨC

- Tầng 0: Tư duy (Decompose, Abstraction, Algorithm)
- Tầng 1: Nền tảng (Syntax, Memory Model, Execution Model)
- Tầng 2: Cấu trúc (Data Structures, Complexity)
- Tầng 3: Thiết kế (SOLID, Design Patterns, System Design)
- Tầng 4: Thực thi (Debugging, Testing, Refactoring)
- Tầng 5: Vận hành (Tooling, Security, Performance, Observability)

## 2. GIAI ĐOẠN PHÁT TRIỂN (CHIỀU DỌC)

### Giai đoạn 1: Xây dựng nền móng (Tầng 1 - 2 - 4)

- Học cú pháp song song với cách bộ nhớ vận hành (Heap/Stack/Pointer).
- Triển khai cấu trúc dữ liệu cơ bản và dùng Debugger để quan sát luồng dữ liệu thực tế.
- Mục tiêu: Code chạy đúng và hiểu bản chất dòng đời của dữ liệu.

### Giai đoạn 2: Chuẩn hóa kỹ năng (Tầng 3 - 4)

- Áp dụng SOLID và Design Patterns để giải quyết bài toán code rối (Spaghetti code).
- Tập trung vào Refactoring và Code Review để nâng cao tính bảo trì và mở rộng.
- Mục tiêu: Code sạch, dễ đọc và linh hoạt khi thay đổi yêu cầu.

### Giai đoạn 3: Tối ưu và Mở rộng (Tầng 5 - 0)

- Nghiên cứu Security, Performance và CI/CD để đưa sản phẩm lên môi trường thực tế.
- Quay lại tầng 0 để rèn luyện khả năng trừu tượng hóa (Abstraction) các bài toán lớn.
- Mục tiêu: Làm chủ hệ thống phức tạp, quy mô lớn và an toàn.

## 3. KẾT NỐI LIÊN TẦNG (CHIỀU NGANG)

Để vượt mức Intermediate, cần liên tục kết nối các node kiến thức:

- Tầng 1 -> Tầng 5: Hiểu Memory Model để tối ưu Performance (tránh memory leak, tận dụng CPU cache).
- Tầng 2 -> Tầng 3: Hiểu đặc tính cấu trúc dữ liệu để thiết kế System Design (chọn loại Database, chiến lược Caching).
- Tầng 3 -> Tầng 4: Dùng Design Patterns làm tiêu chuẩn khi thực hiện Refactoring code.
- Tầng 4 -> Tầng 0: Từ thực tế Debug các hệ thống lớn, đúc kết lại cách chia nhỏ bài toán (Decomposition) hiệu quả hơn.
